# 5.4-beta1

5.4 is intended to be a smaller release mostly bringing compatibility with up to
26.2.

It brings a separate download for 26.x, but you should use the regular
download for world upgrades originating from a different version

## Changelog
- Added compatibility with all versions from 1.21 - 26.2
- Added a separate world download for 26.x
  - Use this for a fresh install in 26.1-snapshot-6 or later
  - Use the regular download to upgrade a world originating from an earlier
    Minecraft version
  - Use this to upgrade a world originating from 26.1-snapshot-6 or later
  - World file is organized in the 26.1-snap6 style
  - Datapack and resourcepack have stripped backwards compatibility with earlier
    versions
- When playing in a 26.2 version, sulfur caves can now appear
- Error messages when opening Fetchr in an unsupported Minecraft version now
  use the full development version name if the version name is in the new style
  (e.g. the 26.x download will now say “This pack has been made compatible for
  Minecraft versions 26.1 Snapshot 6 to 26.2 Release Candidate 2”, whereas it
  is the following for the old format: “This pack has been made compatible for
  Minecraft versions 1.21-1.21.10, or up to snapshot 25w44a”)
  - Since all development versions are now possible to associate with their
    actual version name, this is now possible.
- Adjusted the item pool
- Renamed ”category chests” to “item pool display”
- The item pool chests now have a “Item Pool Display” name in their inventory
- All entities that are connected to another entity connected to the player
  will now be preserved correctly when returning to the lobby

## Lobby Update Changes
There are now two world downloads. The one without a version name in their file
name is compatible with all versions, and **must be** used when updating a
old lobby from a Minecraft version prior to 26.1-snapshot-6, in order to apply
the world format changes to your existing save file from that version.

The world download specifically for 26.x should be used to upgrade an existing
world that originates from 26.1-snapshot-6 or later (which for this first beta
should not happen yet, but is relevant for following versions).

## Item Pool Changes

### Sulfur Caves
Added new sulfur caves category which is available in 26.2-snapshot-1 and later.

**Developer's Note**: _Suflur caves are quite a bit rarer compared to lush caves
and dripstone caves. Feedback on the difficulty of these items is appreciated_

- Sulfur Bricks
- Polished Cinnabar
- Bucket of Sulfur Cube
- Sulfur Spike

### Gold and Ruined Portal
This category has been removed and split into three smaller categories with a
couple of new entries

### Gold

This category has been split from the previous “Gold and ruined portal” category

- Golden Dandelion (new item available in 26.1-snapshot-5 and later)
- Block of Gold
- Block of Raw Gold
- Golden Axe
- Golden Sword
- Golden Shovel

### Gold with Redstone

This category has been split from the previous “Gold and ruined portal” category

- Powered Rail
- Clock

### Ruined Portal

This category has been split from the previous “Gold and ruined portal” category

- Lodestone (previously only with lodestone compass)
- Magma Block
- Obsidian (Previously only with diamond)
- Netherrack (New item; available in all supported Minecraft versions)
- Nether Brick (New item; available in all supported Minecraft versions)

### Books -> Paper
The “Books” category has been renamed to ”Paper”. Empty Map is intentionally not
included.

- Added Name Tag (available in 26.1-snapshot-2 and later)

### Night Time Mob Drops

- Poison arrow is now only available until version 1.21.11 (25w44a)
- Added Weeakness arrow (available in 1.21.11 (25w44a) and later)
  - Also exclusive with Skeleton Drops and Warm Biome items

### Taiga

- Removed Snow from Taiga category.

### Snowy Biomes

- Includes snow
- Includes slowness arrow

### Roots
Removed category and moved all items to new Village category

### Shipwreck loot

- Removed emerald from category
- Added new item “Prismarine Crystals” (available in all Minecraft versions) 

### Village

New Category

- Contains all items from the previous ”Roots” category (carrot and potato-
  related)
- Emerald is now only in the village category
- Added Bell (available in all versions)

**Developer's Note**: _Bell has been previously tested as part of the 5.0
release in 1.16 and has been deemed too uncommon. I feel like villages are more
common in the updated generation. Feedback is appreciated._

## Bug Fixes
- Fixed item pools that featre a category where every item is also part of
  another category breaking card generation if no item is available in that
  category anymore
- Fixed entities getting dismounted during a lobby update
  ([#315](<https://github.com/NeunEinser/bingo/issues/315>))

## Technical Changes
- All internal references in function names, storages, scoreboards, tags,
  translation keys, and other references to “skybox” have now been renamed to
  “spawn_preview”
- All internal references in function names, storages, scoreboards, tags,
  translation keys, and other references to “points_goal” and “points_race” have
  now been renamed to “target_time”
- Updated structure version of `fetchr:tutorial`
  - Translation keys `fetchr.lobby.tutorial.skybox.sign.line<1..4>` and
    `fetchr.lobby.tutorial.skybox.dummy_card_frame.<item_name>` have `skybox`
    changed to `spawn_preview` in their name
  - Translation key `fetchr.lobby.tutorial.chests.lectern.heading` has been
    changed to `fetchr.lobby.tutorial.chests.heading`
- Updated structure version of `fetchr:card_generation`
  - Translation keys `fetchr.settings.points_goal.sign.line1<1..4>`,
    `fetchr.settings.points_goal.lectern.page_<1..2>` and
    `fetchr.settings.points_goal.name` have `points_goal` changed to
    `target_time` in their name
